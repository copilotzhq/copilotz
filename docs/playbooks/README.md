# Playbooks

Playbooks are the primary implementation guides for Copilotz. Each one focuses
on a common application task and shows the recommended primitive, runtime
mapping, and validation path.

## How to Use a Playbook

Each playbook tells you:

- when to use the pattern
- which primitive owns the behavior
- where the files should live
- how Copilotz consumes the resource
- how the behavior appears as runtime or app surface
- how to validate the implementation without guessing
